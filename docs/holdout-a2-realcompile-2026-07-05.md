# Held-out tasks, the A2 grammar arm, and the real compiler — 2026-07-05

Three hardening steps promised by the earlier gates, run in one session:
a 25-task **held-out, human-written** benchmark (bench/tasks-holdout —
indirect phrasings, new domains, distractor scopes, a deprecated trap),
the **A2 grammar-constrained arm** re-run on the tuned model via
llama.cpp, and **real-compiler grading** (`claw defs-check`, which renders
scope stubs + produced defs as a `.claw` module and runs `clawc check`).

Model: Qwen2.5-Coder-0.5B + LoRA retrained on the pod with the committed
corpus-v3 recipe (train loss 0.085; the synthetic gate reproduced at
120/121 — one effects slip vs. the archived 121/121 adapter, i.e. one
task of retrain variance).

## What each layer actually catches

**Holdout drop (100% → 76% clean):** the tuned model generalizes most of
the discipline to human-written tasks but not all of it — 5 hallucination
misses and 2 effect misses on 25 tasks. Base manages 44% here (bigger
scopes make accidental in-scope references likelier), so the fine-tune is
still worth +32 points, but the synthetic 100% does not transfer whole.

**A2 closes references, not semantics.** Under the grammar, holdout clean
rises 76% → 88% and every emitted symbol/effect is structurally legal
(the three A2-holdout misses are truncation/invalid JSON, not
out-of-scope references). But the REAL compiler exposes the other side:
A2's typecheck pass rate craters (23% on synthetic vs. A1's 97%). At
0.5B, constrained decoding pays a "grammar tax" — expression structure
degrades (`add1` emitted a bare `p0` with no lambda; `max3` a contorted
nest of single-parameter lambdas). Constrained references ≠ well-typed
programs.

**The real compiler catches what token-level scoring cannot.** The
holdout A1 outputs that are "clean" by reference-checking still fail
`clawc check` 15/25 times. The failure anatomy (all real examples):

- `area` multiplied the literals 2 and 3 — parameters ignored; every
  reference legal, the program semantically invented.
- `shout` called 2-ary `Str.concat` with one argument (forgot the "!").
- `inc3` called its helper `bump` — which it never defined.

Arity errors, missing siblings, ignored parameters: exactly the class a
typechecker exists for.

## The layered-gate story this proves

1. **CDB context (A1)** kills API hallucination (0% → 100% on synthetic).
2. **Grammar (A2)** makes out-of-scope references *ungeneratable* — the
   backstop when the model is weak or the prompt adversarial — at a
   quality cost that shrinks with model size.
3. **Real typecheck** (`defs-check`) catches arity/undefined/type errors
   the first two layers cannot see.
4. **Contracts** (already wired in the grader) catch what typechecks but
   computes the wrong thing — `area(2,3)` typechecks; `result == w*h`
   does not hold.

No single layer suffices; the pipeline is the product.

## Honest caveats

- 0.5B remains the floor. The grammar tax and the holdout gap should both
  shrink with a larger base model; unverified until we train one.
- The A2 grammar now permits named defs and scope-derived effect rows
  (this session's fix) — but not sibling references, so multi-def tasks
  are structurally unwinnable under the current grammar.
- Real-compile grading maps Nat→U64 and stubs scope symbols with `crash`
  bodies: it checks types, not behavior. Contract execution covers
  behavior only where tasks declare scalar params.
- Holdout was authored by the same project (fresh phrasings, disjoint
  shapes, but not third-party). External contributions welcome —
  bench/tasks-holdout is the place.
