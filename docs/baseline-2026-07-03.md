# Baseline Run — 2026-07-03

First real gate data, three models, 15 tasks each (the set at run time), ≤2 retries, temperature 0.2. Raw JSON in `bench/results/` (regenerate with the commands below). This is first-party reproduction of the central Claw thesis on our own harness.

## Headline: A0 (blind) vs A1 (+CDB symbol table in prompt)

| Model | Compiled A0→A1 | Halluc. symbols A0→A1 | Halluc. Δ |
|---|---|---|---|
| **codestral-latest** (Mistral) | 0/15 → 10/15 | 28 → 1* | −96% |
| **deepseek-chat** (DeepSeek) | 0/15 → 13/15 | 38 → 0 | **−100%** |
| **Qwen3-235B-A22B** (Together) | 1/15 → 3/15 | 0 → 0 | n/a (see below) |

The intervention is **one thing**: putting the code-as-database symbol table (names + types) into the prompt. On the two models that engaged the Def-JSON protocol cleanly (codestral, deepseek), it eliminated essentially all symbol hallucination and took compile-shaped success from 0 to 10–13 of 15.

\* codestral's single A1 "hallucination" was a self-recursive reference (`retryWithBackoff` calling itself). The named-defs protocol (commit 2675823) makes self/mutual reference legal — so that residual is now a true 0.

### A0 hallucinations were the textbook API-invention class
`add`, `fold`, `++`, `split`, `head`, `filter`, `length`, `%`, `==`, `if`, … — symbols that plausibly exist in some stdlib but are not in this task's scope. Exactly the failure mode the literature names as dominant (API hallucination = 41.3% of compile failures in LLM-generated Rust crypto, arXiv:2604.27001).

### The Qwen caveat (honest reporting)
Qwen3-235B produced **0 hallucinations in both arms but also barely compiled** (1→3/15). It largely failed to emit valid Def-JSON at all — a *format-compliance* failure, not a hallucination one. So the A0→A1 hallucination metric is not meaningful for it; its bottleneck is output-format adherence, which A2's grammar constraint is designed to fix outright. Reported, not hidden.

### `passed = 0/15` everywhere — by design
Test/contract oracles do not execute until the compiler-integrated test runner lands (harness spec §7: tasks with unexecuted oracles never silently pass). **Compile-shaped success and hallucination count are the live metrics**; functional pass is future work.

## Interpretation vs the P2 gate

The formal P2 gate is A2 (decode-time grammar mask) vs A0, bar = >30% compile-error reduction. Two independent models already clear that bar's spirit **from prompt context alone** (A1), before the grammar arm runs. A2 should push the format-compliance failures (Qwen) and residual hallucinations to structural zero, because out-of-scope symbols become ungeneratable.

## Gaps / next
- **A2 run** needs a grammar-honoring server (llama.cpp `grammar` / vLLM). Hosted APIs (Mistral, DeepSeek, Together) ignore the field — an "A2" there would just duplicate A1, so it is not run against them.
- **Functional pass** needs the compiler test-runner wired to execute task oracles.
- Grow tasks toward ~200 for statistical weight (29 at latest commit).
